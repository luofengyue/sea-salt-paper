# 《海盐折纸》小程序开发步骤拆解版

> 适用对象：小程序开发小白  
> 使用方式：每次只复制一个步骤里的「给 Codex 的命令」，让 Codex 帮你改代码。  
> 项目目标：先做出一个可以运行、可以单机试玩、后续可扩展好友房的《海盐折纸》小程序原型。  
> 当前策略：先不考虑版权、商业化、随机匹配、复杂美术，优先把游戏跑通。

---

## 0. 项目第一版目标

第一版只做这个闭环：

```text
打开小程序
→ 进入首页大厅
→ 查看规则 / 牌库
→ 开始 AI 练习
→ 完整打一局
→ 结算得分
→ 可以重新开始
```

第一版暂时不做：

```text
随机匹配
排行榜
段位
商城
皮肤
聊天
观战
复杂动画
复杂账号系统
真实多人联机
```

后续再扩展：

```text
创建好友房
→ 邀请朋友
→ 多人联机
→ 同步游戏状态
```

---

## 1. 推荐开发顺序

```text
项目骨架
→ 静态页面
→ 牌库数据
→ 洗牌发牌
→ 玩家摸牌弃牌
→ 弃牌堆拿牌
→ 打出对子
→ 对子效果
→ 喊停
→ 计分结算
→ 简单 AI
→ 体验优化
→ 好友房预留
```

核心原则：

```text
一次只做一个小功能。
每一步做完都要能运行。
不要一开始就做联机。
不要一开始就做复杂美术。
不要同时改太多文件。
```

---

## 2. 给 Codex 下命令的通用格式

以后你可以这样给 Codex：

```text
请基于当前微信小程序项目，完成【某一个小功能】。

要求：
1. 只实现这一个功能，不要扩展其它功能。
2. 保持代码简单，适合小程序新手阅读。
3. 修改完成后告诉我改了哪些文件。
4. 如果需要新增文件，请说明文件作用。
5. 不要引入复杂框架，不要做联机。
6. 保证小程序可以正常编译运行。
```

如果 Codex 一次改太多，你可以补一句：

```text
请缩小范围，只改和这个功能直接相关的文件。
```

## 步骤 12：实现“摸 2 张，选 1 张保留”

### 目标

玩家点击摸牌后，从牌堆摸 2 张，再选择保留其中 1 张。

### 给 Codex 的命令

```text
请帮我实现玩家“摸 2 张，选 1 张保留”的功能。

页面：pages/game/game

流程：
1. 玩家点击“摸牌”。
2. 如果当前 phase 不是 draw，则提示：当前不能摸牌。
3. 从 deck 顶部摸 2 张牌。
4. 把这 2 张牌保存到 tempDrawCards。
5. phase 改为 choose。
6. 页面显示这 2 张临时牌。
7. 玩家点击其中一张作为保留牌。
8. 保留牌加入玩家手牌。
9. 另一张暂时放入 discardPileA。
10. 清空 tempDrawCards。
11. phase 改为 action。

要求：
1. 先默认未保留的牌放入 discardPileA。
2. 暂时不要让玩家选择弃到 A 还是 B。
3. 如果牌堆不足 2 张，先弹出提示：牌堆不足。
4. 不要处理 AI。
```

### 验收标准

```text
点击摸牌后显示 2 张临时牌。
选择 1 张后加入手牌。
另一张进入弃牌堆 A。
phase 从 draw → choose → action。
```
